# Phase 4.5 Working Prototype Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Fix Iris Glass's missing ambient-orb animation, make Iris Glass the default theme with a unified (always-Iris-styled) loading screen, add a floating Settings modal (theme switcher, room/session info, leave chat, about/security), and host the prototype (client on Vercel, relay on Render).

**Architecture:** Builds entirely on top of Phases 1-4's existing crypto/relay/audio layers and theme system — no protocol or crypto changes. A new shared `AmbientOrbs` presentational component backs both the chat screen's and loading screen's orb animation. `App.tsx`'s `Screen` state gains a `safetyNumber` field on its `"chat"` variant so the new `Settings` modal can display it. Hosting is config-only (`render.yaml` + README docs) — no server code changes, since Render's default health check is a plain TCP accept, not an HTTP check.

**Tech Stack:** React 18 + TypeScript + Vite (client), Node + `ws` (server), Vitest (unit tests), Playwright (manual end-to-end visual verification, per the Phase 4 precedent — this codebase has no component-testing library, only Vitest for pure-logic functions).

## Global Constraints

- Commit messages: short, plain-language, imperative mood, human-sounding — no AI-generated verbosity, no bullet-point essays, no `Co-Authored-By` trailers (AGENTS.md rule 4). Never pass `--no-verify` or `--no-gpg-sign`.
- Always run `git commit` via the PowerShell tool, not Bash — Git Bash's bundled `gpg` reads a different keyring and signing silently fails.
- Commit early and often — one commit per task where indicated, not batched at the end.
- Every task must leave `npm run typecheck` and `npm test` (client) green — no task should hand off a red build to the next.
- The loading screen renders in Iris Glass style unconditionally, regardless of the user's selected theme — it does **not** become theme-aware in this phase (that's an explicit roadmap backlog item).
- The theme switcher keeps all three options (Apple, Iris Glass, Pulse Slate) — only the *default* for users with no stored preference changes to Iris Glass.
- The Settings entry point (gear icon) only renders on the chat screen, not on start/waiting/handshake/safety-number screens.
- Settings ships with exactly four sections this phase (theme switcher, room/session info, leave chat, about/security) — do not add more; a roadmap backlog item already covers revisiting scope later.
- No new npm dependencies for the client/server packages (no test-library additions) — Playwright is installed standalone in a scratchpad directory for verification only, per the existing project convention (see Task 7), never added to `client/package.json`.

This task is deliberately one unit, not two: changing the default theme and removing `resolveLoadingScheme` (a theme-system change) only compiles once `LoadingScreen`/`CipherWord`/`App.tsx` stop depending on the removed function and prop — splitting them would hand off a red build mid-way.

**Why `CipherWord` changes:** it currently reads `--font-display` via `getComputedStyle(document.documentElement)` to build the canvas font string for letter-width measurement. `document.documentElement`'s `data-theme` reflects the user's *selected* theme (Apple/Iris/Pulse), not the loading screen's now-fixed Iris Glass style — so if a user has Apple or Pulse selected, the wordmark would silently measure itself with the wrong font and clip, the same class of bug fixed once already in Phase 4 (see `progress.md`). Passing the font family down explicitly removes the dependency on document-root state entirely.

Replace the whole file:

```tsx
import { useMemo } from "react";
import "./CipherWord.css";

const CIPHER_CHARS = "abcdefghijklmnopqrstuvwxyz0123456789";

function randomChar(): string {
  return CIPHER_CHARS[Math.floor(Math.random() * CIPHER_CHARS.length)];
}

function measureWidth(letter: string, font: string): number {
  const canvas = measureWidth.canvas ?? (measureWidth.canvas = document.createElement("canvas"));
  const ctx = canvas.getContext("2d")!;
  ctx.font = font;
  return Math.ceil(ctx.measureText(letter).width);
}
measureWidth.canvas = undefined as HTMLCanvasElement | undefined;

interface CipherWordProps {
  text: string;
  fontSizePx: number;
  startDelayS: number;
  staggerS: number;
  fontFamily: string;
  windowHeightPx?: number;
}

export function CipherWord({
  text,
  fontSizePx,
  startDelayS,
  staggerS,
  fontFamily,
  windowHeightPx = 132,
}: CipherWordProps) {
  const font = `600 ${fontSizePx}px ${fontFamily}`;
  const letters = useMemo(
    () =>
      text.split("").map((letter) => ({
        letter,
        width: measureWidth(letter, font),
        glyphs: [randomChar(), randomChar(), randomChar(), letter],
      })),
    [text, font]
  );

  return (
    <div className="cipher-word" style={{ height: windowHeightPx }}>
      {letters.map((column, index) => (
        <span
          key={index}
          className="cipher-word__column"
          style={{ width: column.width, height: windowHeightPx }}
        >
          <span
            className="cipher-word__reel"
            style={{
              animationDelay: `${startDelayS + index * staggerS}s`,
              height: windowHeightPx * column.glyphs.length,
            }}
          >
            {column.glyphs.map((glyph, glyphIndex) => (
              <span key={glyphIndex} className="cipher-word__glyph" style={{ height: windowHeightPx }}>
                {glyph}
              </span>
            ))}
          </span>
        </span>
      ))}
    </div>
  );
}
```

```tsx
import { ThemeSwitcher } from "../theme/ThemeSwitcher";

  roomCode: string;
  safetyNumber: string;
  onLeave: () => void;

export function Settings({ roomCode, safetyNumber, onLeave, onClose }: SettingsProps) {
```
