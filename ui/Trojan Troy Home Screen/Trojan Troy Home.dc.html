Relay-wake progress bar

Appears on tap of either action. Fills fast, holds alive near the top for cold starts (up to ~60s), completes on the real network event. No copy — the motion carries the meaning.

Handoff tokens & timings
track      height: 4px (thin) | 8px (pill) · radius: 999px · bg: rgba(255,255,255,0.07)
fill       background: linear-gradient(90deg, #6FBF78 0%, #7BC97F 55%, #A6E0A0 100%)
glow       box-shadow: 0 0 8px rgba(123,201,127,.30), 0 0 20px rgba(123,201,127,.13)
drive      width: var --fill % set from JS · transition: width <dur> <ease>
ease       cubic-bezier(0.2, 0.9, 0.3, 1)  — all phases except hold-creep
appear     bar opacity 0 → 1 · 400ms
surge      width 0 → 82% · 1100ms · signature ease
hold       width 82 → 92% · 30000ms · cubic-bezier(0.05, 0.4, 0.2, 1)  (creep)
alive      .sheen 2.4s loop (34% strip, white 0→.34→0, signature ease)
           .breathe 3.2s ease-in-out loop (glow .30 ↔ .52) — separate layers,
           independent of fill %, killed by prefers-reduced-motion
complete   on network event: width → 100% · 500ms · signature ease
settle     glow .60 → .28 · 800ms · signature ease · hold ~350ms
exit       bar opacity → 0 · 400ms, then advance screen
